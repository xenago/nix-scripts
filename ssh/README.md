# ssh

Useful stuff related to SSH.

## Allow RSA host keys to avoid OpenSSH error

On some older gear, an error like this will appear:

`no matching host key type found. Their offer: ssh-rsa`

To get around this temporarily, add it to the runtime command, e.g. `ssh -oHostKeyAlgorithms=+ssh-rsa user@server`

Alternatively, add to `~/.ssh/config`:

    Host nas
      HostName server
      HostKeyAlgorithms=+ssh-rsa

Note: see also [Allow RSA SSH keys to avoid OpenSSH error](#allow-client-rsa-ssh-keys-to-avoid-openssh-error) section below for the server-side configuration.

## SSH Daemon/Service Configuration

Generally, changing `sshd` config is not recommended.

### Enable SSH login as root

Update the configuration file with : `PermitRootLogin yes`, then restart the service.

```sh
nano /etc/ssh/sshd_config
```

Note: you could also create a custom configuration override file instead, e.g. `/etc/ssh/sshd_config.d/custom.conf`

```sh
systemctl restart sshd
```

### Allow client RSA SSH keys to avoid OpenSSH error

OpenSSH generally blocks client RSA keys for security with error  `key type ssh-rsa not in PubkeyAcceptedAlgorithms`.

Create a custom configuration file:
```sh
sudo nano /etc/ssh/sshd_config.d/custom.conf
```

Update the file:
```
HostkeyAlgorithms +ssh-rsa
PubkeyAcceptedAlgorithms +ssh-rsa
```

Restart the service:
```sh
sudo systemctl restart sshd
```

## General commands

### Generate key

Press enter a few times:
```sh
ssh-keygen
```

### Copy public certs to another server

Add all `.pub` files in `~/.ssh` to another server for login:

```sh
ssh-copy-id username@server
```

### Authorize a specific public key file on another server

Notes:
* Use `-f` to force-add the key even if it already exists
* Use `-p 22` to use a different port

```sh
ssh-copy-id -i mykey.pub username@server
```

### Import private key from disk

```sh
cp rsa-private-key-20230321 ~/.ssh/id_rsa
chmod 600 ~/.ssh/id_rsa
```

### Import public key from disk

Works with keys e.g. generated by PuTTY too:

```sh
mkdir ~/.ssh
ssh-keygen -i -f rsa-key-20230321.pub >> ~/.ssh/authorized_keys
```

### Import public key from GitHub

```sh
mkdir ~/.ssh
curl https://github.com/username.keys >> ~/.ssh/authorized_keys
```

### Use another user's account to SSH into a server and run a command

```sh
sudo -su user ssh user@server -f 'ls -lah'
```
